# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

ResidentPravo is the marketing site for a Cyprus-based legal/business consulting firm (company
registration, residence permits ВНЖ/ПМЖ, accounting, business relocation). It is a **multi-page static
site** — 22 HTML pages, no framework, no client-side router. Deployed on Vercel; the deployed build is
generated by a Node build step (see Commands).

Site facts worth knowing when writing copy: primary language Russian, corporate tax 12.5%, founder Artur
Mardanyan, 10+ years practice, clients from 15+ countries. Full canonical facts + page list live in
`llms.txt` at the repo root — keep it in sync when pages are added/changed.

## Commands

```
npm run build       # node scripts/sync-nav.js && node scripts/build-blog-preview.js — runs automatically on Vercel via vercel.json buildCommand
node .dev-server.js # local dev server on http://localhost:3000, applies vercel.json rewrites and mocks /api/contact (logs instead of sending email)
```

There is no lint or test suite. There is no `dev` npm script — run `.dev-server.js` directly.

Deploy: commit and `git push` to the `main` branch tracked at `origin` (Vercel auto-deploys). **Always
confirm with the user before pushing** — this repo has commits that are already ahead of `origin/main`
locally at times.

## Architecture

### URL structure

- Russian is the root language: `/`, `/residence-permit-cyprus`, `/blog`, etc. serve from files under
  `ru/` (plus a few RU files at repo root: `index.html`, `privacy-policy.html`, `terms-of-use.html`).
- English lives under `/en/`, mirroring the same slugs: `en/residence-permit-cyprus.html`, `en/blog/...`,
  `en/experts/...`.
- Pretty URLs (no `.html`) are mapped in `vercel.json` → `rewrites`. **Adding a new page requires adding
  a matching rewrite entry in `vercel.json`**, or it will 404 in production (locally `.dev-server.js`
  reads the same `vercel.json` rewrites, so this is testable before deploying).
- `js/lang-switch.js` maps the current path to its counterpart in the other language for the language
  switcher — it hardcodes which slugs live at EN root vs under `/en/` (`ROOT_EN_PAGES`). New root-level
  EN pages must be added to that list.

### Content areas

- **Service pages** (`{ru,en}/residence-permit-cyprus.html`, `company-registration-cyprus.html`,
  `accounting-audit-cyprus.html`) — long-form service/pillar pages.
- **Blog** (`{ru,en}/blog/index.html` + one file per post, e.g. `cyprus-visas-2026.html`). Posts are
  plain HTML files, not a CMS or markdown pipeline.
- **Expert bios** (`{ru,en}/experts/artur-mardanyan.html`).
- Legal pages (`privacy-policy.html`, `terms-of-use.html`) and `404.html`.

### Build-time codegen (no templating engine — plain HTML + two sync scripts)

Since there's no shared layout/include system, header/nav and blog previews are kept consistent across
all 22 HTML files by scripts that rewrite the static markup in place:

- **`scripts/sync-nav.js`** — single source of truth for nav item order (`Услуги → Почему мы → Процесс →
  Кейсы → Блог → FAQ`, mirrored in EN). Walks every `.html` file and rewrites the nav markup to match.
  Run this after manually editing nav links in any one page, or changes will drift and get overwritten
  on the next build anyway.
- **`scripts/build-blog-preview.js`** — reads `{ru,en}/blog/index.html`, extracts published articles
  (cards with a real date, excluding "Скоро"/"Coming soon" placeholders), sorts newest-first, and injects
  the latest 3 as preview cards into the homepages between `<!-- BLOG:RU:START/END -->` and
  `<!-- BLOG:EN:START/END -->` markers. **New blog posts must be added to `{ru,en}/blog/index.html`
  first** — the homepage preview is derived from it, not edited directly.

Both scripts run automatically on every Vercel build (`npm run build`). When adding a new page or blog
post by hand, re-run `npm run build` locally to catch nav/preview drift before pushing.

### Contact form

`api/contact.js` is a Vercel serverless function (Node, `nodemailer`) that emails submissions to
`residentpravo@gmail.com`. Requires `GMAIL_USER` / `GMAIL_APP_PASSWORD` env vars set in Vercel project
settings. Locally, `.dev-server.js` intercepts `POST /api/contact` and logs the payload instead of
sending real email.

### Other CTAs

Elsewhere on the site, calls-to-action are direct links: `wa.me/357...` (WhatsApp, phone number/message
varies is defined per CTA), `tel:`, and `mailto:` — these don't go through the API.

## SEO — mandatory for all new work

Any new development in this repo — a new page, a new blog post, a new section on an existing page, or a
structural/meta change — **must** go through the `/seo` skill as part of the work, not as an optional
afterthought:

- Before drafting new copy (a page or blog post): run `/seo content-brief` first.
- Before publishing a new/changed page: run `/seo page` for on-page review, and `/seo schema` if it adds
  or changes structured data.
- After adding or changing a page: update `sitemap.xml`, and update `llms.txt` if it changes the
  canonical page list or facts about the business.

This applies even when the user's request doesn't mention SEO explicitly (e.g. "напиши статью про...",
"добавь страницу для...") — treat SEO review as a required step of "done," the same way a build or a
sitemap update is.
